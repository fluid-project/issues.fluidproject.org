    Description

    Comments

    • Michelle D'Souza commented 2008-11-10T08:52:34.000-0500

      We have found that in practice we are simply mapping the values that we use in UI Options to possible values in Access for All. This mapping is currently not very meaningful. We've decided to originally build UI Options without the Access for All bindings and create the bindings once we know more about the values that UI Options uses. This way we can more intelligently feed back our needs to the Access for All spec.
