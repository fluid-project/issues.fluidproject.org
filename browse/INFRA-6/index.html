    • Giovanni Tirloni commented 2015-08-17T14:00:59.150-0400

      I'm able to run the VidyoDashboard VM locally on VirtualBox but it needs direct access to the MySQL database running on the Vidyo appliances. Besides that being a big security risk, the Vidyo appliances are not currently configured to accept external connections to their internal database.

      Another complication is that this VM is not meant to be tweaked and tuned. It runs Ubuntu (not our target platform) and does not allow administrative access. We have to fully trust it, while at the same time it's not clear if there are automatic updates in place, it's not covered by our support contract, etc. So this makes it hard to deploy this VM in our datacenter.

      It's seems we're in a chicken-egg situation: the VM would have to be sitting in the datacenter to access the internal MySQL database, but deploying it in the datacenter is another security risk in itself.

      The VidyoDashboard has proven difficult to integrate in our environment. Additionally, it is not covered by any support contract and does not get security updates. Being a closed box, we cannot modify it to suit our needs.

      There have not been requests for the functionality the VidyoDashboard provides since a long time ago. It seems we have not run into issues where VidyoDashboard would be useful (although it does provide nicer reports).

      Closing issue for now until there is more demand and/or Vidyo improves the situation regarding how VidyoDashboard is supported.
