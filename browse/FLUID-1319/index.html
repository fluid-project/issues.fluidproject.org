    • Antranig Basman commented 2008-11-07T13:30:56.000-0500

      This has been fixed at revision 5965 by simply unconditionally catching the exception. Note that this issue is investigated and characterised more fully as part of GWT in the following thread:

      http://code.google.com/p/google-web-toolkit/issues/detail?id=1385

      The upshot is that the error occurs when attempting to setSelectionRange on any text control which is not currently "physically embodied" in front of the user, e.g. hidden as a result of being markup running in a test ase.
