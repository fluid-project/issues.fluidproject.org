    • James Yoon commented 2010-02-23T15:35:08.000-0500

      This is an interesting issue where we have to balance the usability between sighted and VO users.

      For the VO user, my feeling is that we should allow the description to be fully readable (whether it's visually collapsed or expanded), and prevent swiping over the "more" and "less" buttons/links. The fact that the description can be fully read without surrendering screen real estate is an affordance VO users have over other users.

      But I can see a boundary use case where the device's primary user is a VO user, and he/she has listened to the description and wants to show it to his/her sighted friend to read. How, then, would the VO user expand the description for his/her friend?

      The middle-of-road solution would be to force VO users to read a truncated description (just like sighted users), and expand it to read the rest (again, just like sighted users).
