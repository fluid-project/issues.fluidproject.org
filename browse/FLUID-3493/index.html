    Description

    Implementing a template with a number of empty divs on Opera seems to expose a parsing anomaly whereby not all close tags are seen (guestbook.html in Engage). Also, a possibly related issue is seen on Firefox with duplication of empty <br/> tags

    Comments

    • Antranig Basman commented 2010-02-09T20:38:43.000-0500

      This has been revealed to be an issue of corruption of empty tags (such as <div />) through the document HTML. In Opera, these are quietly degraded to "<div>" with resulting corruption of the document structure. Either do not write templates with these tags, or else armour them with CDATA or comment wrapping in order to protect them.

    • Justin Obara commented 2014-07-07T15:38:17.942-0400

      Opera is no longer a supported browser.
