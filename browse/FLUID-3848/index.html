    • Harris Wong commented 2010-11-30T12:14:16.954-0500

      Justin and I tried adding aria-sort into the pager demo, but NVDA-main-3946 and JAWS 12.0 didn't report the sort strategy (ascending/descending) after you click on the column with a mouse or keyboard. We have looked into the actual accessibility DOM tree with Accessibility Probe, there is a "sort=ascending" property in the attribute within ia2. This means that the aria-sort are reflected on FF3.6 but the screen readers are not picking it up. We believe it is a problem with the screen readers and we are waiting for a reply from them.

    • Harris Wong commented 2010-11-30T15:13:00.781-0500

      Includes aria-sort in the column headers after the sort action is triggered. Note that the pager sorter only uses ascending and descending sort order, thus the "other" option of the aria-sort is not used. W3 WAI spec can be found here, http://www.w3.org/WAI/PF/aria/states_and_properties#aria-sort.

    • Harris Wong commented 2010-12-06T14:52:40.709-0500

      • Patch includes changes to the anchor within the sortableHeaders, anchor now have 3 different titles depending on the sort direction.
