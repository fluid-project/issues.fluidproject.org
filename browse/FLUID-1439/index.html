    Description

    Several inconsistencies in cursor styling are caused by the fact that we are currently causing cursor changes over grab handles using manual JS event handlers. As well as being inefficient, this makes it hard to impossible to catch all of the many cases where a cursor movement ought to be registered as over the grab handle.

    However, applying these hover rules via CSS will require some discipline and documentation, as well as presenting a strong implementation risk under IE6, which does not allow CSS hover styles to be implemented on non-a elements.
