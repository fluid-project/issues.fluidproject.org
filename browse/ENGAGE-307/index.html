    • Colin Clark commented 2010-02-19T00:20:04.000-0500

      I have committed a modified version of the patch that Yura and Justin submitted on this ticket. A couple of things to note in the future, Yura/Justin/Sveto:

      1. Don't forget the Same Origin Policy: you can't POST via AJAX in the browser to a different server
      2. Writing a service that will blindly POST to any URL specified in the query parameters is a security flaw
      3. HTTP semantics matter. Don't GET when you should be POSTing

      I modified the server-side code to read the sendMyCollectionURL out of Engage's config file, rather than accepting it as a request parameter. I have not yet changed the client-side code to do a POST instead of a GET. That'll have to happen for 0.3 final.

      I'm also not fully convinced that a jQuery UI Dialog is the most optimal solution here. It works, but it's going to be slow on a mobile phone. Given Safari's lack of support for most ARIA roles, it may also not be the most accessible option.
