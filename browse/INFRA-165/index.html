    Comments

    • Giovanni Tirloni commented 2018-04-24T13:55:10.135-0400

      • Moved all websites from /srv/www to /var/www (this is more aligned with CentOS/RHEL SELinux defaults)
      • Reset all SELinux settings back to factory defaults
      • Switch from permissive to enforcing mode
      • Enabled the `httpd_can_network_connect_db` boolean so php-fpm can connect to MySQL/PostgreSQL
      • Configured the following file contexts for Wordpress/Drupal/Joomla:
      semanage fcontext -a -t httpd_sys_rw_content_t '/var/www(/.*)?/tmp(/.*)?'
semanage fcontext -a -t httpd_sys_rw_content_t '/var/www(/.*)?/wp-content(/.*)?'
semanage fcontext -a -t httpd_sys_rw_content_t '/var/www(/.*)?/cache(/.*)?'

      Due to these changes, updating website code through their UI will not work anymore because php-fpm doesn't have permission to write to anything besides the 3 directories listed above (even if traditional Unix permissions allow it).

      Affected servers: i-0013 and i-0018
