    • Jacob Farber commented 2009-03-26T11:31:28.000-0400

      The more and more complex our css options become, the more bugs we're seeing in how browsers handle large repetetive document repainting tasks. In this case, once I focus an element it immediately repaints and reflows accordingly, but not before.

      This could symptomatic of another CSS issue, but it looks like we may need to employ some sort of "forced repaint/reflow" for browsers.
