    Description

    There appears to be some tests failing randomly in the all-tests.html. However, when they are run individually they pass. Different tests are failing on different runs, so it is hard to track down what is causing it exactly. It seems that it could be the case where elements that are needed for the test are destroyed before the test completes.

    IE 8 (Windows 7)

    • Antranig Basman commented 2014-06-11T10:16:01.875-0400

      This issue results from a timing issue (race condition) in the IoC testing framework. I believe it will require an upgrade to qunit in order to resolve it but I haven't had time to investigate it in detail. I have seen these failures on several browsers and not just IE8, especially if the machine is running slowly. The workaround is to run the failing tests individually and verify that they run. We should push this out for Infusion 2.0 since it will take a lot of diagnosis to resolve.
