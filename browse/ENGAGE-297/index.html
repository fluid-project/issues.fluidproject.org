    • Michelle D'Souza commented 2010-03-10T18:25:34.000-0500

      The one change in this patch that seems a little odd is the format of the json data files and therefore the expectation of initComponentWithLocalData. The data files now contain an options block rather then just a model. Yura explained that the reason he made that change was because the exhibition view requires a model for the guestbook sub component. This model needs to be specified as part of the exhibition view options not as part of the exhibition view model. If we don't change the expectations of initComponentWithLocalData then we would need to put demo data into the options we pass to it. This defeats the purpose of separating out the demo data from the html file.

      2. merge the options passed into initComponentWithLocalData with the options returned from the json data files and update the API docs accordingly

    • y z commented 2010-03-10T18:58:12.000-0500
