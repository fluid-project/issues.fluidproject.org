    Description

    Currently the staging site built from the main looks identical to the floe stories production site. In fact the floe stories site uses the staging site to demo the storytelling tool. However, this is problematic because they look identical but when you change to the staging links to the browser stories are for those that are written in the staging site but not those from the floe stories site. It's confusing for a user.

    We should probably have a "nightly" build site deployed from main. The nightly should include all of the features like users accounts and etc. Each published site should have their own staging sites so that updates to them can be tested without effecting the published site.
