    • Laurel Williams commented 2009-08-12T15:39:24.000-0400

      Further examination yielded the following suggested solution from Colin:

      1. Write a script for the server that will invoke Maven correctly, with an explicit path
      to the settings.xml file.
      2. Parameterize the command that gets executed in our Ant build script to run Maven
      3. Set a default for this value that works for most people's machines
      4. In the PHP script that runs Ant (which runs Maven; whew!), specify the alternative,
      server-specific script instead of mvn directly

      The benefit here is that we make only a tiny impact on our build script--no duplicate
      code and users of the script will never need to know.

    • Laurel Williams commented 2009-08-25T14:59:25.000-0400

      The script recommended by Colin is now working. I have not committed the script since it consists of a single line of code and includes directories specific to our particular server. An example of the code is:

      #!/bin/sh

      1. A script which calls maven pointing to the correct maven configuration file - used by web user only.
      2. execute mvn --settings filename install
        mvn --settings /home/myhome/.m2/settings.xml install

      Created a file called customBuild.sh and put this text in it. Saved the file in the same directory as the pom.xml (main infusion directory). Made the script executable by all (since the web user needs to access it).

      The ant script patches I submitted earlier run this script to access maven from the browser interface.
