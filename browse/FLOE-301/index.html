    Description

    Implement a sticky simulator so that a user can experiment with sticky keys from within a web interface.

    It should be possible to apply the simulator to a text entry field. If a user presses a modifier key, it should be "remembered" and applied to the subsequent keypress in the text entry field.

    press modifier once - added to the subsequent character input only
    press modifier twice - added to all subsequent character inputs
    press modifier three times - disables the sticky key

    e.g.

    • User places focus in a textfield.
    • User presses "shift"
    • User presses "a"
    • The textfield outputs "A"

    • Simon Bates commented 2015-04-22T13:21:05.786-0400

      After some discussions, the following changes have been made to the sticky keys implementation:

      • sticky keys is implemented for shift only
      • the text input on which sticky keys can be activated shows at most 1 character of input (subsequent input replaces the character, if there is one)
      • we do not implement locking of the shift key, only latching for a single input
      • pressing shift toggles the shift latch status (that is, pressing the shift key twice without any other input will turn the shift latching off, rather than lock)
      • shift is unlatched after a shifted key is input
      • we have added support for an indicator to show when the shift key is latched – this is done by setting a class on the text input whenever latching is active (see /tests/manualTests/html/keyboardInput-manualTests.html for an example)

    • Simon Bates commented 2015-04-22T13:49:24.494-0400

      In addition, due to the nature of the information and functionality available within the browser, the shifting behaviour is US keyboard specific.
