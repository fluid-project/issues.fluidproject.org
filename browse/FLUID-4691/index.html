    Description

    When using jQuery 1.7.1, if you have tooltips switched on with a RichTextEditor on the page and hover over content, that content will disappear, which means you can no longer edit it.

    Comments

    • Antranig Basman commented 2012-04-04T22:14:48.329-0400

      I couldn't reproduce this issue once I upgraded to a matched set of jQuery UI and jQuery 1.7.2 - however, I did find that the rich text integration was broken completely as a result of an API change in jQuery for the $.html() function which has suffered the same kind of malign "drift" that affected $.val() in jQuery 1.5.1 - this now explicitly counts its arguments rather than being satisfied to see whether they have a value of "undefined" or not. I have pushed up a version with a fixed framework shim that seems to work for me at
      https://github.com/amb26/infusion/tree/FLUID-4684
      Please test and let me know how it seems to you.
