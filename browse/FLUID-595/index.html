    Description

    On OSDPL pages that aren't part of a category (taxonomy), in the page body the word "in" appears by itself. This is confusing, and appears because this page is not "in" any category. This occurs on the home page as well as News pages, or any other page which doesn't have a category. I'm not sure which module is creating this "in" and am hoping there is a configuration option somewhere that can just turn this off.
