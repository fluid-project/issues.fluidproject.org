    • Justin Obara commented 2010-07-19T12:48:01.013-0400

      It appears that this issue is caused by the ctrlKey property on the event object not setting the value to true on keydown. Rather this property is set to true on keyup.

      This causes the conditional on line 217 of Reorderer.js to fail because isMove(evt) returns false.

    • Justin Obara commented 2010-07-19T15:38:57.185-0400

      Some further investigation shows that the behaviour displayed in linux is opposite of that elsewhere. In firefox 3.6 on Mac OS 10.6 for example, keydown has ctrlKey set to true, but keyup is false

    • Jonathan Hung commented 2010-12-09T15:38:41.135-0500

      This issue is partially fixed. Mouse dragging styles now apply properly. However, keyboard actions still require two key presses to initiate.
